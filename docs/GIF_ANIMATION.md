# GIF Анимация эрозии

## Обзор

Функционал GIF анимации реализован в соответствии с планом DEVELOPMENT_PLAN.md (Phase 2, Priority 1) и позволяет визуализировать динамику эрозии береговой линии во времени.

## Реализация

**Модуль:** `internal/cli/gif_export.go`
**Формат:** GIF89a, 800x600 пикселей
**Библиотека:** стандартная `image/gif` (без внешних зависимостей)

## Использование

### Базовая анимация

```bash
./lito model erosion --steps 5 --output-gif erosion.gif
```

### С настройкой FPS

```bash
./lito model erosion --steps 10 --output-gif fast.gif --gif-fps 15
```

### С пропусканием кадров (для уменьшения размера)

```bash
./lito model erosion --steps 20 --output-gif reduced.gif --gif-skip 2
```

### С временной динамикой

```bash
./lito model erosion \
  --steps 8 \
  --target-years 20 \
  --years-per-step 2.5 \
  --storm-probability 0.3 \
  --output-gif temporal.gif
```

### Комплексная модель

```bash
./lito all \
  --iterations 3 \
  --steps 10 \
  --output-gif full.gif \
  --gif-fps 12 \
  --gif-skip 1
```

## Параметры CLI

### `--output-gif <path>`
- **Описание:** Путь к GIF файлу для анимации эрозии
- **По умолчанию:** пустое (GIF отключен)
- **Пример:** `--output-gif erosion.gif`

### `--gif-fps <1-30>`
- **Описание:** Кадров в секунду для GIF анимации
- **По умолчанию:** 10
- **Диапазон:** 1-30 FPS
- **Пример:** `--gif-fps 15`

### `--gif-skip <N>`
- **Описание:** Пропускать каждый N-ный кадр для уменьшения размера файла
- **По умолчанию:** 1 (не пропускать)
- **Пример:** `--gif-skip 2` (использовать каждый 2-й кадр)

## Технические детали

### Алгоритм работы

1. **Сбор снимков:** Система собирает snapshots береговой линии на каждом шаге эрозии
2. **Фильтрация:** Опциональное пропускание кадров для уменьшения размера
3. **Проекция:** Географические координаты проецируются на 2D изображение
4. **Отрисовка:** Каждый кадр рисуется с палитрой optimized для научной визуализации
5. **Компиляция:** GIF создается с настроенной задержкой между кадрами

### Палитра цветов

- **0 (20,40,60):** Глубокая вода (темно-синий)
- **1 (30,80,120):** Мелководье (синий)
- **2 (46,139,87):** Суша/берег (зеленый)
- **3 (139,69,19):** Внутренняя суша (коричневый)
- **4 (255,255,255):** Текст (белый)

### Размеры файлов

Типичные размеры GIF файлов:
- **5 кадров, 5 FPS:** ~19KB
- **6 кадров (с пропуском), 15 FPS:** ~23KB
- **9 кадров с временной динамикой, 8 FPS:** ~29KB

## Ограничения

1. **Разрешение:** Фиксированное 800x600 пикселей
2. **Палитра:** Ограничена 10 цветами для оптимизации размера
3. **Производительность:** Генерация больших GIF (>100 кадров) может занять время
4. **Память:** Каждый кадр хранится в памяти при генерации

## Troubleshooting

### GIF не создается
- Убедитесь, что указан `--output-gif` параметр
- Проверьте права записи в директорию вывода
- Убедитесь, что моделирование создает достаточно снимков (>1)

### GIF слишком большой
- Используйте `--gif-skip` для уменьшения числа кадров
- Уменьшите число шагов моделирования
- Снизьте FPS если это не критично

### GIF слишком медленный/быстрый
- Настройте `--gif-fps` параметр
- Типичные значения: 5-15 FPS для научной визуализации
