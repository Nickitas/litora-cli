# Улучшенная GIF анимация с цветовыми кодами ✅

## Обзор

Реализована улучшенная GIF анимация для визуализации эрозии береговой линии с **цветовым кодированием** и **высоким качеством**. Теперь GIF анимации действительно полезны для научных публикаций и анализа.

## Что улучшено

### 🎨 Цветовое кодирование
- **Зеленый градиент:** слабая эрозия → умеренная → сильная
- **Красный градиент:** очень сильная → экстремальная эрозия
- **Голубой градиент:** аккумуляция наносов (осаждение)
- **Серая линия:** начальное состояние береговой линии

### 📐 Улучшенное качество
- **Разрешение:** 1200x800 пикселей (было 800x600)
- **Палитра:** 15 цветов для научной визуализации (было 10)
- **Размер файла:** 45-60KB для качественной анимации

### 📊 Метрики на кадрах
- Номер кадра / общее количество
- Текущая длина береговой линии
- Накопленная эрозия
- Эрозия за шаг

## Использование

### Базовая анимация с цветовыми кодами (по умолчанию)
```bash
./lito model erosion --steps 8 --output-gif colored.gif
```

### С настройками качества
```bash
./lito model erosion \
  --steps 10 \
  --output-gif high_quality.gif \
  --gif-fps 12 \
  --gif-color-change \
  --gif-show-initial \
  --gif-show-metrics
```

### Отключить цветовое кодирование
```bash
./lito model erosion \
  --steps 6 \
  --output-gif simple.gif \
  --gif-color-change=false
```

### С временной динамикой
```bash
./lito model erosion \
  --steps 8 \
  --target-years 20 \
  --years-per-step 2.5 \
  --storm-probability 0.3 \
  --output-gif temporal_colored.gif
```

## Новые CLI флаги

### `--gif-color-change` (по умолчанию: true)
Включает цветовое кодирование сегментов по интенсивности эрозии/аккумуляции.

### `--gif-show-initial` (по умолчанию: true)
Показывает начальное состояние береговой линии тонкой серой линией для сравнения.

### `--gif-show-metrics` (по умолчанию: true)
Добавляет метрики на каждый кадр (длина, эрозия, номер кадра).

## Цветовая легенда

**Цвета эрозии (берег отступает):**
- 🟢 Светло-зеленый: слабая эрозия (< 5% от максимума)
- 🟡 Желто-зеленый: умеренная эрозия (5-20%)
- 🟠 Желто-оранжевый: средняя эрозия (20-40%)
- 🟠 Оранжевый: сильная эрозия (40-60%)
- 🔴 Красно-оранжевый: очень сильная эрозия (60-80%)
- 🔴 Ярко-красный: экстремальная эрозия (> 80%)

**Цвета аккумуляции (берег растет):**
- 🔵 Светло-голубой: слабая аккумуляция
- 🔵 Голубой: умеренная аккумуляция
- 🔵 Синий: сильная аккумуляция

## Технические детали

### Алгоритм цветового кодирования
1. Анализ изменений каждого сегмента между начальным и конечным состоянием
2. Нормализация изменений относительно максимального значения
3. Присвоение цвета на основе интенсивности и направления (эрозия/аккумуляция)
4. Отрисовка сегментов с соответствующими цветами

### Улучшения качества
1. **Высокое разрешение:** 1200x800 вместо 800x600
2. **Расширенная палитра:** 15 специально подобранных цветов
3. **Толстые линии:** 2 пикселя для лучшей видимости
4. **Научная палитра:** градиенты оптимизированы для печати и презентаций

### Интеграция с CSV
```bash
./lito model erosion \
  --steps 10 \
  --output-gif analysis.gif \
  --output-csv metrics.csv
```
GIF визуализация + CSV данные для количественного анализа

## Преимущества для научных публикаций

1. **Визуализация пространственной изменчивости:** чётко видно, где эрозия максимальна
2. **Сравнение сценариев:** цветовые коды позволяют легко сравнивать разные модели
3. **Презентации:** высокое качество подходит для конференций и статей
4. **Количественный анализ:** метрики на кадрах облегчают интерпретацию
5. **Временная динамика:** видно как меняется интенсивность процессов

## Ограничения и рекомендации

### Размер файла
- Базовый: 45-60KB для 8-10 кадров при 1200x800
- Оптимально: 5-15 кадров для научных презентаций
- Для более длинных анимаций используйте `--gif-skip`

### Производительность
- Генерация: ~1-2 секунды на кадр при 1200x800
- Для быстрых тестов используйте меньшее разрешение или меньше кадров

### Научные рекомендации
- Используйте цветовое кодирование для анализа пространственной изменчивости
- Добавляйте сравнение с начальным состоянием для наглядности
- Включайте метрики для количественной интерпретации
- Используйте временные метки для анализа динамики
