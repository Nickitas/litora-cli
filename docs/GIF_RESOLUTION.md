# Настраиваемое разрешение GIF (Custom Resolution) ✅

## Обзор

Реализована настройка разрешения GIF анимации для разных вариантов использования: научные публикации, презентации, веб и социальные сети.

## Новые CLI флаги

- `--gif-width` - ширина GIF в пикселях (по умолчанию: 1200)
- `--gif-height` - высота GIF в пикселях (по умолчанию: 800)

## Рекомендуемые разрешения

### 📊 Для научных публикаций
```bash
--gif-width 1200 --gif-height 800  # стандартное (40-50KB)
--gif-width 1920 --gif-height 1080 # высокое качество (60-70KB)
--gif-width 2400 --gif-height 1600 # очень высокое качество (80-100KB)
```

### 🎯 Для презентаций
```bash
--gif-width 1600 --gif-height 900  # 16:9 (50-60KB)
--gif-width 1920 --gif-height 1080 # Full HD (60-70KB)
--gif-width 1280 --gif-height 720  # HD (45-55KB)
```

### 🌐 Для веба
```bash
--gif-width 800 --gif-height 600   # стандартное (25-30KB)
--gif-width 640 --gif-height 480   # компактное (20-25KB)
--gif-width 1024 --gif-height 768  # среднее (35-40KB)
```

### 📱 Для социальных сетей
```bash
--gif-width 1080 --gif-height 1080 # квадратное Instagram (40-50KB)
--gif-width 1200 --gif-height 675  # Twitter 16:9 (35-45KB)
--gif-width 800 --gif-height 600   # компактное (25-30KB)
```

## Примеры использования

### Высокое качество для статьи:
```bash
./lito model erosion \
  --steps 12 \
  --target-years 30 \
  --output-gif figure4_high_res.gif \
  --gif-width 2400 \
  --gif-height 1600 \
  --gif-geo-labels all \
  --gif-colorlegend-pos right
```

### Стандартное для презентации:
```bash
./lito model erosion \
  --steps 10 \
  --output-gif presentation.gif \
  --gif-width 1920 \
  --gif-height 1080 \
  --gif-scalebar-km 50
```

### Компактное для веба:
```bash
./lito model erosion \
  --steps 8 \
  --output-gif web_animation.gif \
  --gif-width 800 \
  --gif-height 600 \
  --gif-skip 2
```

### Квадратное для Instagram:
```bash
./lito model erosion \
  --steps 6 \
  --output-gif instagram_post.gif \
  --gif-width 1080 \
  --gif-height 1080 \
  --gif-colorlegend-pos bottom
```

## Зависимость размера файла от разрешения

### Тестовые данные (3 шага эрозии):
- 640×480 = 21KB (веб, быстрая загрузка)
- 800×600 = 27KB (компактное)
- 1080×1080 = 43KB (квадратное)
- 1200×800 = 40KB (стандартное)
- 1600×900 = 53KB (презентационное)
- 1920×1080 = 65KB (Full HD)

### Оптимизация размера:
- Используйте `--gif-skip N` для пропуска кадров
- Меньшее разрешение = меньше размер файла
- Квадратные форматы эффективны для социальных сетей
- 16:9 формат оптимален для презентаций

## Выбор разрешения для задач

### Научная статья (высокое качество):
- 2400×1600 для детальных графиков
- 1920×1080 для стандартных фигур
- Включить все элементы: scale bar, color legend, geo labels

### Конференция (среднее качество):
- 1920×1080 для Full HD проекторов
- 1600×900 для стандартных проекторов
- Читаемые метки и масштабная линейка

### Веб-публикация (оптимизированное):
- 800×600 для быстрой загрузки
- 640×480 для мобильных устройств
- Минимальные элементы для чистого вида

### Социальные сети (специальные форматы):
- 1080×1080 для Instagram (квадрат)
- 1200×675 для Twitter (16:9)
- Яркие цвета и четкие метки

## Технические детали

### Автоматическое масштабирование
- При изменении разрешения автоматически масштабируются все элементы
- Scale bar адаптируется к новому размеру
- Текстовые метки сохраняют пропорции

### Соотношение сторон
- Стандартное: 3:2 (1200×800)
- Презентационное: 16:9 (1920×1080)
- Квадратное: 1:1 (1080×1080)
- Классическое: 4:3 (800×600, 1024×768)

### Память и производительность
- Большее разрешение = больше памяти при генерации
- Время генерации ~ линейно зависит от разрешения
- Оптимальный баланс: 1200×800 (40-50KB)
