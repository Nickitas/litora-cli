## Алгоритм Рамера — Дугласа — Пекера

Алгоритм Рамера — Дугласа — Пекера (Ramer-Douglas-Peucker, RDP) является стандартным методом упрощения полилиний с сохранением существенной формы кривой.

### Математическая основа

**Цель упрощения:** Сократить число точек, сохранив форму кривой в пределах заданного допуска (tolerance).

**Ключевая идея:** Рекурсивно находить точку с максимальным отклонением от отрезка между концами. Если отклонение > допуска — сохранить точку и рекурсивно обработать обе половины.

```
Исходная:    A────────────────────────────B

Шаг 1: найти точку P с max отклонением от AB

Если |P, AB| > tolerance:
    A────────P────────────B
    (сохранить P, рекурсия на AP и PB)

Иначе:
    A────────────────────B
    (удалить все промежуточные)
```

### Рекурсивный алгоритм упрощения

**Структура данных:**

```go
type SimplifyOptions struct {
    MaxPoints int // Целевое максимальное число точек
}

type SimplifyResult struct {
    Points           []LatLon // Упрощённые точки
    OriginalCount    int      // Исходное число точек
    SimplifiedCount  int      // Число точек после упрощения
    ToleranceMeters  float64  // Найденный допуск в метрах
    Applied          bool     // Было ли применено упрощение
    OriginalClosed   bool     // Была ли исходная замкнутой
    SimplifiedClosed bool     // Осталась ли замкнутой
}
```

**Рекурсивный алгоритм:**

```go
func markSimplifiedPoints(projected []pointXY, keep []bool, start, end int, toleranceSquared float64) {
    if end-start < 2 {
        return
    }
    
    // Найти точку с максимальным отклонением
    maxDistance := -1.0
    index := -1
    for i := start + 1; i < end; i++ {
        distance := squaredSegmentDistance(projected[i], projected[start], projected[end])
        if distance > maxDistance {
            maxDistance = distance
            index = i
        }
    }
    
    // Если отклонение в пределах допуска → завершить ветвь
    if index == -1 || maxDistance <= toleranceSquared {
        return
    }
    
    // Сохранить точку
    keep[index] = true
    
    // Рекурсия на две половины
    markSimplifiedPoints(projected, keep, start, index, toleranceSquared)
    markSimplifiedPoints(projected, keep, index, end, toleranceSquared)
}
```

**Пример работы:**

```
Исходные точки:  [A, B, C, D, E, F, G, H]
                 A────────────────────────────H

Итерация 1:      A───────────G──────────────H  (max отклонение в G)
                  сохранить: [A, G, H]

Итерация 2:      A──────D────G                (левая половина)
                  A──────────────H                (правая половина)
                  сохранить: [A, D, G, H]

Результат:        [A, D, G, H]  (50% reduction)
```

### Расстояние от точки до отрезка

**Расстояние от точки P до отрезка AB:**

```go
func squaredSegmentDistance(point, a, b pointXY) float64 {
    dx := b.X - a.X
    dy := b.Y - a.Y
    
    // Если A и B совпадают
    if dx == 0 && dy == 0 {
        return squaredDistance(point, a)
    }
    
    // Параметрическая координата проекции
    t := ((point.X - a.X) * dx + (point.Y - a.Y) * dy) / (dx*dx + dy*dy)
    
    switch {
    case t <= 0:
        // Ближе к A
        return squaredDistance(point, a)
    case t >= 1:
        // Ближе к B
        return squaredDistance(point, b)
    default:
        // Проекция на отрезок
        projection := pointXY{
            X: a.X + t * dx,
            Y: a.Y + t * dy,
        }
        return squaredDistance(point, projection)
    }
}
```

**Геометрическая интерпретация:**

```
        P
        •
       ↕ 
      ╱ ╲
     ╱   ╲  ← расстояние перпендикуляра (0 < t < 1)
    ╱     ╲
   A───────B
   ↑       ↑
t ≤ 0     t ≥ 1
```

### Бинарный поиск допуска

**Проблема:** Пользователь задаёт целевое число точек (`MaxPoints`), но алгоритм RDP работает с допуском в метрах.

**Решение:** Бинарный поиск наилучшего допуска:

```go
func SimplifyPolyline(points []LatLon, options SimplifyOptions) SimplifyResult {
    // ... инициализация ...
    
    projected := projectToMeters(points)
    diagonal := projectedDiagonal(projected) // Размер bounding box
    
    low := 0.0         // Минимальный допуск
    high := diagonal    // Максимальный допуск
    best := clonePoints(points)
    
    // 24 итерации бинарного поиска
    for i := 0; i < 24; i++ {
        mid := (low + high) / 2
        
        simplified := simplifyWithTolerance(points, projected, mid)
        
        if len(simplified) > target {
            low = mid // Нужно строже
        } else if len(simplified) < minPoints {
            high = mid // Нужно мягче
        } else {
            best = simplified
            high = mid // Попробовать строже
        }
    }
    
    return SimplifyResult{
        Points:          best,
        ToleranceMeters: bestTolerance,
        Applied:         true,
    }
}
```

**Почему 24 итерации?**

```
precision = diagonal / 2^24 ≈ diagonal / 1.67 × 10^7
```

Для диагонали 1000 км это ~60 метров точности — достаточно для практических целей.

### Обработка замкнутых полилиний

**Особый случай:** Замкнутые полилинии (первая и последняя точки совпадают) требуют специальной обработки.

```go
closed := isClosedPolyline(points)

if closed {
    if len(points) <= 4 {
        return result // Слишком мало точек для упрощения
    }
    
    // Временно убрать замыкающую точку
    working := clonePoints(points[:len(points)-1])
    target := options.MaxPoints - 1 // Зарезервировать место для замыкания
    minPoints := 3 // Минимум треугольник
    
    // Упрощение без замыкающей точки
    simplified := SimplifyPolyline(working, SimplifyOptions{MaxPoints: target})
    
    // Добавить замыкающую точку обратно
    result.Points = append(simplified.Points, simplified.Points[0])
}

result.SimplifiedClosed = true
```

**Пример:**

```
Исходная (замкнутая): [A, B, C, D, E, F, A]  (6 уникальных + замыкание)
                       ╔═════════╗
                       ║ A       B ║
                       ║         ║
                       ║ F       C ║
                       ║         ║
                       ║ E       D ║
                       ╚═════════╝

После упрощения:    [A, C, E, A]  (3 уникальных + замыкание)
                     ╔───╗
                     ║ A ║─┐
                     ╚───╝ │
                           │
                     ┌─────┘
                     │
                   E═┘
```

## IDW интерполяция

**IDW (Inverse Distance Weighting)** — метод интерполяции, при котором значения в ближайших точках имеют больший вес, чем в удалённых.

### Обратная дистанционная весовая функция

**Математическая основа:**

Для оценки значения в точке `(lat, lon)` по N ближайшим точкам измерения:

```
wᵢ = 1 / distanceᵢ^p

Wᵢ = wᵢ / Σwⱼ

V = Σ(Wᵢ × vᵢ)

где:
  wᵢ — необработанный вес точки i
  p — степень (обычно p = 2)
  Wᵢ — нормированный вес точки i
  V — интерполированное значение
  vᵢ — измеренное значение в точке i
```

**Влияние степени p:**

| Степень p | Влияние расстояния | Применение |
|-----------|-------------------|------------|
| p = 1 | Линейное убывание | Мягкая интерполяция |
| p = 2 | Квадратичное убывание | Стандартный вариант |
| p > 2 | Быстрое убывание | Резкая локализация |

### Алгоритм интерполяции

**Структура данных для литологии:**

```go
type LithologyProfile struct {
    Metadata  LithologyMetadata         // Метаданные профиля
    Points    []LithologyPoint          // Точки замера литологии
    Classes   map[string]LithologyClass // Классы пород
}

type LithologyState struct {
    Class       string  // Класс породы
    Resistance  float64 // Интерполированное сопротивление
    Color       string  // Цвет для визуализации
    Description string  // Описание
}
```

**Алгоритм IDW интерполяции:**

```go
func (p *LithologyProfile) GetLithologyAt(lat, lon float64) LithologyState {
    const maxPoints = 6     // Максимальное число точек для интерполяции
    const power = 2.0       // Степень для IDW
    
    // Найти ближайшие точки
    nearby := p.findNearbyPoints(lat, lon, maxPoints)
    if len(nearby) == 0 {
        return p.getDefaultLithology()
    }
    
    // Рассчитать веса IDW
    weights := make([]float64, len(nearby))
    weightSum := 0.0
    
    for i, point := range nearby {
        dist := math.Sqrt(math.Pow(point.Lat - lat, 2) + math.Pow(point.Lon - lon, 2))
        
        if dist < 1e-6 {
            // Точное совпадение → используем эту точку
            return LithologyState{...point...}
        }
        
        // IDW вес: 1 / distance^power
        weights[i] = 1.0 / math.Pow(dist, power)
        weightSum += weights[i]
    }
    
    // Нормировать веса
    for i := range weights {
        weights[i] /= weightSum
    }
    
    // Интерполировать resistance (взвешенное среднее)
    interpolatedResistance := 0.0
    for i, point := range nearby {
        interpolatedResistance += weights[i] * point.Resistance
    }
    
    // Для класса и цвета — выбор по максимальному весу
    maxWeightIdx := 0
    for i := range weights {
        if weights[i] > weights[maxWeightIdx] {
            maxWeightIdx = i
        }
    }
    
    dominantPoint := nearby[maxWeightIdx]
    
    return LithologyState{
        Class:       dominantPoint.Lithology,
        Resistance:  interpolatedResistance,
        Color:       dominantPoint.Color,
        Description: dominantPoint.Description,
    }
}
```

### Выбор числа соседей

**Компромисс между точностью и эффективностью:**

| N соседей | Точность | Эффективность | Применение |
|-----------|----------|---------------|------------|
| 1-3 | Низкая (зависит от случайных расположений) | Высокая | Быстрые оценки |
| 4-6 | Хорошая (баланс локальности и гладкости) | Хорошая | Стандартный вариант |
| 7-10 | Высокая (гладкая интерполяция) | Средняя | Высококачественная интерполяция |
| >10 | Маргинальный прирост | Низкая | Редко используется |

**Стандартное значение:** `maxPoints = 6` оптимально для большинства задач.

### Гибридная интерполяция качественных и количественных данных

**Проблема:** Литологические данные включают как количественные (сопротивление), так и качественные (класс породы, цвет) характеристики.

**Решение:** Гибридный подход

```go
// Количественные данные (сопротивление) — взвешенное среднее
interpolatedResistance := Σ(weights[i] × point.Resistance)

// Качественные данные (класс, цвет) — выбор по максимальному весу
maxWeightIdx := argmax(weights)
dominantClass := nearby[maxWeightIdx].Lithology
dominantColor := nearby[maxWeightIdx].Color
```

**Логика:** Ближайшая точка доминирует для качественных характеристик, но все точки влияют на количественные значения.

## Билинейная интерполяция

**Билинейная интерполяция** используется для интерполяции значений в регулярной сетке (grid), что особенно важно для батиметрических данных.

### Regular Grid архитектура

**Структура регулярной сетки:**

```go
type BathymetryGrid struct {
    Points     map[string]BathymetryPoint // Карта точек сетки
    Resolution float64                     // Размер ячейки (градусы)
    bounds                                 // Границы сетки
}

type BathymetryPoint struct {
    Lat   float64 // Широта левого верхнего угла ячейки
    Lon   float64 // Долгота левого верхнего угла ячейки
    Depth float64 // Глубина в метрах (отрицательная)
}
```

**Индексация ячеек:**

```go
func gridKey(lat, lon, resolution float64) string {
    latIdx := math.Floor(lat / resolution)
    lonIdx := math.Floor(lon / resolution)
    return fmt.Sprintf("%s,%s", latIdx, lonIdx)
}
```

**Пример сетки:**

```
Разрешение: 0.01° (~1.1 км)

Ячейки (lat, lon):
    [45.00, 30.00] → "4500,3000" → depth: -100 м
    [45.00, 30.01] → "4500,3001" → depth: -150 м
    [45.01, 30.00] → "4501,3000" → depth: -120 м
    [45.01, 30.01] → "4501,3001" → depth: -180 м
```

### Интерполяция в ячейке

**Билинейная интерполяция по 4 соседним углам ячейки:**

```
P00 •------• P01
    |      |
    |  *   |  (*) — запрашиваемая точка
    |      |
P10 •------• P11

depth = (1-t)(1-u)×P00 + t(1-u)×P01 + (1-t)u×P10 + t×u×P11

где:
  t = (lon - lon₀) / resolution  // Нормированная координата X
  u = (lat - lat₀) / resolution  // Нормированная координата Y
```

**Алгоритм интерполяции:**

```go
func (g *BathymetryGrid) InterpolateDepth(lat, lon float64) (float64, error) {
    // Проверка границ
    if lat < g.bounds.MinLat || lat > g.bounds.MaxLat ||
       lon < g.bounds.MinLon || lon > g.bounds.MaxLon {
        return 0, fmt.Errorf("координаты вне границ сетки")
    }
    
    // Найти углы ячейки
    lat0 := math.Floor(lat / g.Resolution) * g.Resolution
    lon0 := math.Floor(lon / g.Resolution) * g.Resolution
    lat1 := lat0 + g.Resolution
    lon1 := lon0 + g.Resolution
    
    // Получить 4 угловых точки
    p00 := g.Points[gridKey(lat0, lon0, g.Resolution)]
    p01 := g.Points[gridKey(lat0, lon1, g.Resolution)]
    p10 := g.Points[gridKey(lat1, lon0, g.Resolution)]
    p11 := g.Points[gridKey(lat1, lon1, g.Resolution)]
    
    // Нормированные координаты
    fx := (lon - lon0) / g.Resolution
    fy := (lat - lat0) / g.Resolution
    
    // Билинейная интерполяция
    i0 := bilinearInterpolate1D(p00.Depth, p01.Depth, fx)
    i1 := bilinearInterpolate1D(p10.Depth, p11.Depth, fx)
    depth := bilinearInterpolate1D(i0, i1, fy)
    
    return depth, nil
}

func bilinearInterpolate1D(v0, v1, t float64) float64 {
    return v0 + t * (v1 - v0)  // Линейная интерполяция
}
```

### Валидация границ

**Проверка координат:**

```go
func validateBathymetryPoints(points []BathymetryPoint) error {
    const (
        minLat = 40.0
        maxLat = 47.0
        minLon = 27.0
        maxLon = 42.5
        margin = 0.1  // Допуск на граничные эффекты
        maxDepth = -3000.0  // Максимальная глубина Чёрного моря
    )
    
    for i, p := range points {
        // Проверка координат с tolerant margin
        if p.Lat < minLat-margin || p.Lat > maxLat+margin {
            return fmt.Errorf("широта вне диапазона Чёрного моря")
        }
        if p.Lon < minLon-margin || p.Lon > maxLon+margin {
            return fmt.Errorf("долгота вне диапазона Чёрного моря")
        }
        
        // Проверка глубины
        if p.Depth > 0 {
            return fmt.Errorf("глубина должна быть отрицательной")
        }
        if p.Depth < maxDepth {
            return fmt.Errorf("глубина превышает реалистичную")
        }
    }
    
    return nil
}
```

**Валидация сетки:**

```go
func validateBathymetryGrid(grid *BathymetryGrid) error {
    if len(grid.Points) == 0 {
        return fmt.Errorf("сетка пуста")
    }
    
    if grid.Resolution <= 0 {
        return fmt.Errorf("некорректное разрешение")
    }
    if grid.Resolution > 0.1 {
        return fmt.Errorf("слишком грубое разрешение (max 0.1°)")
    }
    
    return nil
}
```

### Эффективность поиска

**Сложность операций:**

| Операция | Сложность | Описание |
|----------|-----------|------------|
| `gridKey(lat, lon)` | O(1) | Вычисление индекса ячейки |
| `Points[key]` | O(1) | Поиск точки в hash map |
| `InterpolateDepth` | O(1) | Интерполяция (4 точки) |

**Преимущества Regular Grid:**
- ✅ O(1) доступ к данным
- ✅ Простая интерполяция

## Линейная интерполяция полилиний

Линейная интерполяция используется для аппроксимации сегментов полилиний и проекции географических координат в метрическую систему.

### Проекция географических координат

**Задача:** Конвертация географических координат (lat, lon) в декартову систему (метры) для геометрических операций.

**Алгоритм проекции:**

```go
func projectToMeters(points []LatLon) []pointXY {
    // Вычислить среднюю точку как опорную
    refLat := mean(points[i].Lat for i in points)
    refLon := mean(points[i].Lon for i in points)
    
    // Метры на градус
    metersPerDegLat := 111194.9
    metersPerDegLon := metersPerDegLat * cos(refLat × π/180)
    
    // Защита от полюсов
    if |metersPerDegLon| < 1e-9:
        metersPerDegLon = metersPerDegLat
    
    // Проекция всех точек
    projected := make([]pointXY, len(points))
    for i, point := range points {
        projected[i] = pointXY{
            X: (point.Lon - refLon) × metersPerDegLon,
            Y: (point.Lat - refLat) × metersPerDegLat,
        }
    }
    
    return projected
}
```

**Формулы:**

```
x = (lon - refLon) × metersPerDegLon
y = (lat - refLat) × metersPerDegLat

где:
  metersPerDegLat = 111194.9 м
  metersPerDegLon = 111194.9 × cos(refLat × π/180)
```

### Аппроксимация сегментов

**Аппроксимация сегмента полилинии отрезком:**

```go
// Для сегмента между соседними точками
tangent := {
    X: next.X - prev.X,
    Y: next.Y - prev.Y,
}

// Нормализация
tangentLength := √(tangent.X² + tangent.Y²)
tangent := {
    X: tangent.X / tangentLength,
    Y: tangent.Y / tangentLength,
}

// Нормали (перпендикуляры)
leftNormal := {-tangent.Y, tangent.X}
rightNormal := {tangent.Y, -tangent.X}
```

## Сравнительный анализ методов

### Сравнение методов интерполяции

| Метод | Точность | Эффективность | Применение | Преимущества | Недостатки |
|-------|----------|---------------|------------|---------------|------------|
| **IDW** | Средняя-высокая | O(N) | Литология, нерегулярные данные | Гибкость, простота | Чувствительность к выбору p |
| **Билинейная** | Высокая | O(1) | Батиметрия, регулярные сетки | Скорость, гладкость | Требует регулярную сетку |
| **Линейная** | Базовая | O(1) | Проекции, аппроксимации | Простой, детерминистский | Ограниченная точность |

### Сравнение методов упрощения

| Метод | Сохранение формы | Скорость | Контроль точности | Применение |
|-------|-----------------|----------|------------------|------------|
| **RDP** | Отличное | O(N log N) | Прямой (tolerance) | Упрощение полилиний |
| **Grid sampling** | Хорошее | O(N) | Косвенный (разрешение) | Растеризация |
| **Random sampling** | Плохое | O(N) | Нет | Быстрые оценки |

### Рекомендации по выбору метода

**Выбор метода интерполяции:**

```
IF данные на регулярной сетке:
    Использовать билинейную интерполяцию
ELSE IF данные нерегулярны, но плотные:
    Использовать IDW с 4-6 соседями
ELSE IF данные разрежены:
    Использовать IDW с 2-3 соседями + высокую степень p
ELSE:
    Использовать ближайшего соседа
```

**Выбор метода упрощения:**

```
IF нужна высокая точность формы:
    Использовать RDP с малым tolerance
IF допустима потеря деталей:
    Использовать RDP с большим tolerance
IF нужна максимальная скорость:
    Использовать grid sampling
```

## Практические рекомендации

### Настройка параметров RDP

**Выбор tolerance:**

| Применение | Tolerance | Результат |
|------------|-----------|-----------|
| SVG рендеринг | 100-500 м | Визуально неотличимо, быстрая отрисовка |
| Анализ формы | 10-50 м | Сохранение деталей |
| Статистический анализ | 500-1000 м | Грубое упрощение |

**Выбор MaxPoints:**

```
MaxPoints = √(N_original) × 2  // Эмпирическая формула

Примеры:
  N_original = 1000   → MaxPoints ≈ 63
  N_original = 10000  → MaxPoints ≈ 200
  N_original = 100000 → MaxPoints ≈ 632
```

### Настройка параметров IDW

**Выбор степени p:**

```
p = 1.0  → мягкая интерполяция, хорошие глобальные тренды
p = 2.0  → стандартный выбор, баланс локальности и гладкости
p = 3.0+ → резкая локализация, точная близкая интерполяция
```

**Выбор числа соседей:**

```
N = 4-6   → стандартный выбор для большинства задач
N = 2-3   → для очень плотных данных
N = 7-10  → для высококачественной интерполяции
```

### Настройка параметров Regular Grid

**Выбор разрешения:**

```
Resolution = 0.01°  → ~1.1 км (региональный масштаб)
Resolution = 0.001° → ~110 м (локальный масштаб)
Resolution = 0.0001° → ~11 м (детальный масштаб)
```

**Рекомендации для Чёрного моря:**

| Масштаб | Разрешение | Применение |
|---------|------------|------------|
| Региональный | 0.01° (1.1 км) | Общий анализ, волновая эрозия |
| Локальный | 0.001° (110 м) | Детальная батиметрия |
| Локальный (высокий) | 0.0005° (55 м) | Прибрежная зона |

### Диагностика проблем

**Проблемы интерполяции и решения:**

| Симптом | Возможная причина | Решение |
|---------|-------------------|---------|
| Остроконечные артефакты | Мало соседей в IDW | Увеличить число соседей |
| Размытые границы | Слишком много соседей | Уменьшить число соседей |
| Потеря деталей | Слишком большой tolerance RDP | Уменьшить допуск |
| Медленная работа | Неэффективный выбор метода | Использовать Regular Grid вместо IDW |

Методы аппроксимации обеспечивают баланс между точностью представления геометрических данных и эффективностью их обработки, что критически важно для геоморфологического моделирования береговых линий.
