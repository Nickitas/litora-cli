# Описание модели эрозии береговой линии

## Общая архитектура

В системе Litoro-cli реализована комбинированная эрозионная модель, включающая два принципиально разных подхода к моделированию динамики береговой линии:

1. **Гауссова стохастическая эрозия** — базовая модель случайных деформаций
2. **Волновая физически обоснованная эрозия** — продвинутая модель с учётом гидродинамики

## 1. Гауссова стохастическая эрозия

### Математическая основа

Модель применяет случайные смещения к каждой точке контура береговой линии с нормальным распределением Гаусса:

```
dx ~ N(0, σ²)
dy ~ N(0, σ²)
```

где `σ` — параметр `strength` (стандартное отклонение в метрах)

### Алгоритм реализации

**Этап 1: Геодезическая конверсия**
```go
refLat = mean(latitudes)
metersPerDegLat = 111194.9 м
metersPerDegLon = 111194.9 × cos(refLat)
```

**Этап 2: Генерация случайных смещений**
```go
dx = rng.NormFloat64() × strength
dy = rng.NormFloat64() × strength
```

**Этап 3: Применение смещений**
```go
newLat = oldLat + dy / metersPerDegLat
newLon = oldLon + dx / metersPerDegLon
```

### Особенности реализации

1. **Замкнутые контура:** первая и последняя точки смещаются идентично для сохранения топологии
2. **Детерминизм:** поддержка фиксированного seed для воспроизводимости результатов
3. **Параллельность:** разбиение на чанки по 512 точек для эффективной многопоточной обработки
4. **Геодезическая точность:** учёт широтно-зависимой шкалы долгот

### Параметры модели

| Параметр | Тип | Описание | Значение по умолчанию |
|----------|-----|----------|---------------------|
| `strength` | float64 | Стандартное отклонение смещения (м) | - |
| `seed` | int64 | Сид генератора случайных чисел | time.Now().UnixNano() |

## 2. Волновая физически обоснованная эрозия

### Концептуальная основа

Моделирует реальную физику береговых процессов на основе:
- **Волновой гидродинамики** — энергия волн зависит от длины разгона
- **Экспозиции берега** — угол падения волн относительно береговой нормали
- **Батиметрии** — глубина моря модулирует волновую энергию
- **Локальной геометрии** — выступы и бухты эродируют по-разному

### Математическая модель

#### 2.1. Определение морской стороны

Для каждой точки вычисляются две нормали:
```go
tangent = normalize(next - prev)
leftNormal = (-tangent.y, tangent.x)
rightNormal = (tangent.y, -tangent.x)
```

Морская нормаль выбирается по максимальному волновому score:
```go
seawardNormal = argmax(leftResponse, rightResponse)
```

#### 2.2. Волновой экспозиционный анализ

Функция `sampleWaveSide` вычисляет интегральный score волнового воздействия:

**Многонаправленное семплирование:**
```go
directions = sampleWaveDirection(mainDirection, spread, samples, count)
```
- Генерируется 9 направлений в диапазоне ±55° от основного направления ветра
- Каждое направление взвешивается по косинусу угла падения

**Ray-casting для fetch distance:**
```go
fetch = rayFetchDistance(projected, index, direction, closed, probeDist, maxFetch)
```
- Луч испускается вдоль направления волны
- Находится пересечение с другими сегментами берега
- Максимальная длина разгона ограничена 150 км

**Вычисление факторов:**
```go
fetchFactor = √(meanFetch / maxFetch)
exposure = Σ(incidence^exposurePower) / samples
depthFactor = 1 - exp(-effectiveDepth / depthScale)
```

**Итоговый score:**
```go
score = fetchFactor × exposure × (0.35 + 0.65 × depthFactor)
```

#### 2.3. Батиметрическая модуляция

Функция `physicalDepthFactor` моделирует затухание волн на мелководье:

```go
depthFactor = 1 - exp(-effectiveDepth / depthScale)
```

где:
- `effectiveDepth = max(0, -depthMeters)` (глубина всегда положительна)
- `depthScale = 4000 м` (характерный масштаб затухания)

**Физический смысл:**
- Глубокая вода (> 12 км) → `depthFactor ≈ 1` → полная волновая энергия
- Мелководье (< 1 км) → `depthFactor ≈ 0.22` → сильное затухание

#### 2.4. Геометрическая модуляция

Локальная геометрия берега модулирует скорость эрозии:

```go
shapeDelta = midpoint(prev, next) - current
localScale = 0.5 × (distance(current, prev) + distance(current, next))

protrusion = clamp(-shapeDelta · seawardNormal / localScale, 0, 1.5)
bayShelter = clamp(shapeDelta · seawardNormal / localScale, 0, 1.2)
```

**Интерпретация:**
- `protrusion > 0` — выступ в море → ускоренная эрозия
- `bayShelter > 0` — бухта → защищённость от волн

#### 2.5. Вычисление величины отступания

```go
windFactor = (windSpeed / 12)²  [0.1, 4.0]
retreat = strength × windFactor × score × (0.55 + protrusion - 0.35 × bayShelter)
```

**Ограничения:**
```go
retreat = min(retreat, maxRetreat)
retreat = max(retreat, 0)
```

#### 2.6. Сглаживание и стохастика

**Сглаживающий компонент:**
```go
smoothingAlpha = min(retreat / localScale, 0.5)
newPoint = current - seawardNormal × retreat + shapeDelta × smoothingAlpha
```

**Нерегулярность:**
```go
retreat ×= clamp(1 + N(0, σ_irregularity), 0.7, 1.3)
```

### Параметры волновой эрозии

| Параметр | Обозначение | Значение | Физический смысл |
|----------|-------------|----------|-----------------|
| Скорость ветра | `WindSpeedMetersPerSecond` | 12 м/с | Базовая скорость ветра |
| Направление ветра | `WindSourceDirectionDeg` | 0-360° | От севера по часовой |
| Разброс волн | `FetchSpreadDeg` | ±55° | Диапазон направлений |
| Число семплов | `FetchSamples` | 9 | Количество направлений |
| Макс. разгон | `MaxFetchMeters` | 150 км | Предельная длина fetch |
| Масштаб глубины | `DepthScaleMeters` | 4 км | Характерная глубина |
| Степень экспозиции | `ExposurePower` | 1.5 | Степень в функции |
| Макс. отступание | `MaxRetreatMeters` | =strength | Ограничение сдвига |
| Дистанция пробы | `ProbeDistanceMeters` | 25 м | Отступ для ray casting |
| Нерегулярность | `Irregularity` | 0.25 | Коэффициент случайности |

## 3. Интеграция с батиметрическими данными

### Структура батиметрической сетки

```go
type BathymetryGrid struct {
    Points     map[string]BathymetryPoint
    Resolution float64  // ~0.01° для Чёрного моря
    bounds     struct {
        MinLat, MaxLat float64  // [40.0, 47.0]°N
        MinLon, MaxLon float64  // [27.0, 42.0]°E
    }
}
```

### Билинейная интерполяция

Для произвольных координат (lat, lon):

```go
lat0 = floor(lat / resolution) × resolution
lon0 = floor(lon / resolution) × resolution

depth = bilinearInterpolate(
    depth(lat0, lon0), depth(lat0, lon1),
    depth(lat1, lon0), depth(lat1, lon1),
    (lon - lon0) / resolution,
    (lat - lat0) / resolution
)
```

### Graceful degradation

При отсутствии батиметрических данных используется геометрический proxy:

```go
depthFactor = 1 - exp(-normalFetch / depthScale)
```

Это обеспечивает работоспособность модели даже без реальных глубин.

## 4. Вычислительная оптимизация

### Параллельная обработка

```go
chunkSize = 512 точек
for chunk in chunks(points, chunkSize):
    parallel_process(chunk, seed)
```

Каждая точка получает детерминированный случайный сдвиг:
```go
localSeed = seed + step×10000 + index
```

### Проектирование координат

Для ускорения геометрических вычислений:

```go
projected = metersFromLatLon(points)
reference = mean(points)
// Все вычисления в метрах, затем обратная проекция
```

## 5. Валидация и ограничения

### Географические ограничения (Чёрное море)

```go
lat ∈ [40.0°, 47.0°]
lon ∈ [27.0°, 42.0°]
depth ∈ [-3000 м, 0 м]
```

### Численные ограничения

```go
0.1 ≤ windFactor ≤ 4.0
0 ≤ retreat ≤ maxRetreat
-1.5 ≤ protrusion ≤ 1.5
0 ≤ bayShelter ≤ 1.2
```

## 6. Применимость модели

### Сильные стороны

1. **Физическая обоснованность** — учёт реальной гидродинамики
2. **Локальная детерминированность** — зависит от конкретной геометрии
3. **Масштабируемость** — параллельная обработка больших контуров
4. **Гибкость** — параметризация для разных регионов

### Ограничения

1. **Квази-2D приближение** — не учитывает вертикальную эрозию
2. **Стационарная батиметрия** — глубины не обновляются со временем
3. **Отсутствие седиментации** — не моделируется перенос наносов
4. **Линейная суперпозиция** — не учитывает нелинейные эффекты

### Рекомендации по использованию

- **Калибровка:** параметр `strength` требует подстройки под конкретный регион
- **Валидация:** сравнение с историческими данными об эрозии
- **Комбинация:** использование вместе с фрактальным анализом для комплексной оценки
