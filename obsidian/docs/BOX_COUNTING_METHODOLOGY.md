# Методология box-counting для фрактального анализа

## Общее описание

Метод box-counting является стандартным алгоритмом для оценки эмпирической фрактальной размерности геометрических объектов. В системе Litoro-cli он реализован с рядом улучшений для повышения стабильности и воспроизводимости результатов.

## Теоретическая основа

### Фрактальная размерность

Фрактальная размерность D характеризует how detail increases with resolution:

```go
N(ε) ~ (1/ε)^D
```

где:
- `N(ε)` — число непустых ячеек размера `ε`
- `D` — фрактальная размерность

В логарифмических координатах:

```go
log(N(ε)) = D × log(1/ε) + const
```

### Метод box-counting

Алгоритм покрытия объекта квадратной сеткой с переменным размером ячейки:

1. Покрыть объект сеткой с ячейками размера ε
2. Посчитать число непустых ячеек N(ε)
3. Повторить для различных ε
4. Оценить D как наклон линейной регрессии log(N(ε)) от log(1/ε)

## Реализация в Litoro-cli

### Архитектурные улучшения

В отличие от базового метода, реализация в Litoro-cli включает три ключевых улучшения:

#### 1. Усреднение по смещениям сетки

```go
gridOffsets = [][2]float64{
    {0, 0},     // базовая сетка
    {0.5, 0},   // сдвиг на ½ ячейки по X
    {0, 0.5},   // сдвиг на ½ ячейки по Y
    {0.5, 0.5}, // сдвиг на ½ по обеим осям
}
```

**Обоснование:** Базовый метод чувствителен к произвольному положению сетки. Усреднение по четырём смещениям снижает этот эффект.

#### 2. Плотный набор масштабов

```go
scaleFactors = [4, 6, 8, 12, 16, 24, 32, 48, 64, 96, 128, 192, 256]
```

**Обоснование:** Более плотное покрытие масштабного диапазона улучшает качество регрессии и устойчивость оценки.

#### 3. Адаптивный выбор окна регрессии

Вместо использования всех масштабов выбирается наиболее стабильный диапазон:

```go
window = bestRegressionWindow(logInvScale, logBoxes)
```

**Критерии выбора:**
- Минимальная длина окна: 4 масштаба
- Минимальный R²: 0.98
- Максимальный разброс локальных наклонов: 0.18

### Детальный алгоритм

#### Этап 1: Проецирование в метры

```go
func latLonToMeters(p LatLon) Point2D:
    const refLat = 43.5° // центр Чёрного моря
    const metersPerDegLat = 111194.9 м
    const metersPerDegLon = 87300.0 м
    
    dLat = (p.Lat - refLat) × metersPerDegLat
    dLon = (p.Lon - 35.0) × metersPerDegLon
    
    return Point2D{X: dLon, Y: dLat}
```

#### Этап 2: Вычисление bounding box

```go
minX, maxX, minY, maxY = bboxMeters(points)
bboxSize = max(maxX - minX, maxY - minY)
```

#### Этап 3: Box-counting с усреднением

Для каждого масштаба:

```go
func boxesCoveredMetersAverage(points, boxSize, offsets):
    totalBoxes = 0
    
    for offset in offsets:
        covered = empty_set
        
        for each segment (points[i-1], points[i]):
            markSegmentBoxesOffset(covered, segment, boxSize, offset)
        
        totalBoxes += size(covered)
    
    return totalBoxes / len(offsets)
```

**Разметка сегмента:**

```go
func markSegmentBoxesOffset(covered, a, b, boxSize, offset):
    dx = b.X - a.X
    dy = b.Y - a.Y
    distance = hypot(dx, dy)
    steps = ceil(distance / (boxSize/2)) + 1
    
    for i from 0 to steps:
        t = i / steps
        x = a.X + dx × t
        y = a.Y + dy × t
        
        row = floor((y - minY + offsetY × boxSize) / boxSize)
        col = floor((x - minX + offsetX × boxSize) / boxSize)
        
        covered.add((row, col))
```

#### Этап 4: Построение выборки для регрессии

```go
samples = []
for factor in scaleFactors:
    boxSize = bboxSize / factor
    boxes = boxesCoveredMetersAverage(points, boxSize, gridOffsets)
    
    if boxes > 1:
        sample = {
            ScaleFactor: factor,
            BoxSizeMeters: boxSize,
            BoxesCovered: boxes,
            LogInvScale: log(1 / (boxSize / bboxSize)),
            LogBoxes: log(boxes)
        }
        samples.append(sample)
```

#### Этап 5: Выбор оптимального окна регрессии

**Перебор всех возможных окон:**

```go
func bestRegressionWindow(x, y):
    best = null
    
    for start from 0 to n-minScaleSamples:
        for end from start+minScaleSamples-1 to n-1:
            window = x[start:end+1], y[start:end+1]
            
            slope, intercept = linearRegression(window.x, window.y)
            r2 = regressionRSquared(window.x, window.y, slope, intercept)
            localSlopes = localSlopeSeries(window.x, window.y)
            spread = valueSpread(localSlopes)
            
            stable = (len(localSlopes) >= 3) and 
                     (r2 >= 0.98) and 
                     (spread <= 0.18)
            
            if betterWindow(best, currentWindow, stable):
                best = currentWindow
    
    return best
```

**Критерии сравнения окон:**

```go
func betterWindow(current, candidate, candidateStable):
    // Стабильность — приоритет №1
    if candidateStable != currentStable:
        return candidateStable
    
    // Длина окна — приоритет №2
    if candidate.length != current.length:
        return candidate.length > current.length
    
    // Качество регрессии — приоритет №3
    if |candidate.r2 - current.r2| > 1e-9:
        return candidate.r2 > current.r2
    
    // Для близких R² — детализация (наклон)
    if |candidate.r2 - current.r2| <= 1e-3:
        if |candidate.slope - current.slope| > 1e-6:
            return candidate.slope > current.slope
    
    // Разброс локальных наклонов — приоритет №4
    return candidate.spread < current.spread
```

#### Этап 6: Локальная диагностика

```go
func localSlopeSeries(x, y):
    slopes = []
    for i from 1 to len(x)-1:
        denominator = x[i] - x[i-1]
        if |denominator| > 1e-12:
            slopes.append((y[i] - y[i-1]) / denominator)
    return slopes
```

#### Этап 7: Итоговая оценка

```go
finalAnalysis = {
    Dimension: window.slope,
    RegressionRSquared: window.r2,
    StableAcrossScales: stable,
    StabilitySpread: spread,
    Samples: allSamples,
    LocalDimensions: localSlopes,
    Valid: (0.5 <= slope <= 3.0)
}
```

## Параметры метода

### Константы качества

```go
const (
    minScaleSamples       = 4   // минимальное число масштабов
    minStableLocalSlopes  = 3   // минимальное число локальных наклонов
    minRegressionRSquared = 0.98// минимальное качество регрессии
    maxLocalSlopeSpread   = 0.18// максимальный разброс наклонов
)
```

### Масштабный диапазон

```go
// Относительные масштабы (от размера bbox)
scaleFactors = [4, 6, 8, 12, 16, 24, 32, 48, 64, 96, 128, 192, 256]

// Абсолютные размеры ячеек (при bbox ~500 км)
boxSizes ≈ [125, 83, 62, 42, 31, 21, 16, 10, 7.8, 5.2, 3.9, 2.6, 2.0] км
```

### Смещения сетки

```go
// 4 положения для усреднения
gridOffsets = [
    (0, 0),     // исходное положение
    (0.5, 0),   // ½ ячейки по X
    (0, 0.5),   // ½ ячейки по Y
    (0.5, 0.5)  // ½ по обеим осям
]
```

## Диагностика качества

### Индикаторы надёжности

1. **Коэффициент детерминации R² ≥ 0.98**
   - Высокое линейное соответствие в лог-лог координатах
   - Indicates good scaling behavior

2. **Стабильность локальных наклонов spread ≤ 0.18**
   - Низкая вариативность наклонов между соседними масштабами
   - Указывает на фрактальный режим

3. **Достаточная длина окна ≥ 4 масштабов**
   - Статистическая значимость регрессии
   - Надёжная оценка наклона

4. **Физический диапазон размерности 0.5 ≤ D ≤ 3.0**
   - Исключение артефактов и вырожденных случаев

### Типичные проблемы

#### Проблема: Нестабильность оценки

**Симптомы:**
- Большой разброс локальных наклонов (>0.18)
- Низкий R² (<0.98)
- Разные оценки при смещении сетки

**Причины:**
- Недостаточное разрешение данных
- Вырожденная геометрия (слишком простая/сложная)
- Плохой выбор масштабного диапазона

**Решение в Litoro-cli:**
- Усреднение по 4 смещениям сетки
- Адаптивный выбор стабильного окна
- Плотный набор масштабов

#### Проблема: Масштабная зависимость

**Симптомы:**
- D меняется при изменении масштабного диапазона
- Нет стабильного окна с хорошими показателями

**Причины:**
- Мультифрактальность объекта
- Переход между режимами (гладкий ↔ фрактальный)
- Смешение геометрий разных масштабов

**Решение в Litoro-cli:**
- Диагностика локальных наклонов
- Выбор наиболее стабильного диапазона
- Отчёт о масштабной зависимости

## Сравнение с теоретическими значениями

### Классическая кривая Коха

**Теоретическое значение:**
```go
D_koch = log(4) / log(3) ≈ 1.261859507...
```

**Эмпирическая оценка Litoro-cli:**
- Для органической кривой Коха с Seed=42
- При 5-6 итерациях: D ≈ 1.26 ± 0.03

**Вывод:** Метод корректно восстанавливает теоретическое значение в пределах погрешности.

### Природные береговые линии

**Характерные диапазоны:**
- Чёрное море: D ≈ 1.05-1.15
- Мировой океан: D ≈ 1.02-1.30
- Высоко фрагментированные: D ≈ 1.20-1.50

**Физическая интерпретация:**
- D ≈ 1.0 — гладкая линия (евклидов случай)
- D ≈ 1.1-1.2 — умеренная фрактальность (типично для берегов)
- D ≈ 1.3-1.5 — высокая фрактальность (выраженная изрезанность)

## Ограничения метода

### Методические ограничения

1. **Чувствительность к выбору масштабов**
   - Решено: адаптивный выбор окна + плотный набор масштабов

2. **Зависимость от положения сетки**
   - Решено: усреднение по 4 смещениям

3. **Конечность реальных данных**
   - Неизбежно: верхний масштабный предел ограничен размером объекта
   - Нижний предел ограничен разрешением данных

4. **Анизотропия геометрии**
   - Береговые линии inherently анизотропны
   - Box-counting с квадратными ячейками усредняет эффект

### Интерпретационные ограничения

1. **Не является доказательством фрактальности**
   - Высокий R² указывает на масштабную инвариантность
   - Но не гарантирует истинной фрактальности

2. **Эффективная размерность ≠ внутренняя размерность**
   - Box-counting измеряет эффективную (емкостную) размерность
   - Может отличаться от других определений (корреляционная, информация)

3. **Масштабная ограниченность**
   - Фрактальные свойства могут проявляться только в определённом диапазоне
   - Вне этого диапазона геометрия может быть евклидовой

## Рекомендации по использованию

### Для научных исследований

1. **Всегда проверяйте стабильность:**
   - R² ≥ 0.98
   - spread ≤ 0.18
   - Достаточная длина окна

2. **Сравнивайте с теорией:**
   - Известные фракталы (Кох, Sierpiński) для валидации
   - Природные объекты для калибровки

3. **Анализируйте масштабную зависимость:**
   - Посмотрите на локальные наклоны
   - Проверьте наличие разрывов/переходов

4. **Документируйте параметры:**
   - Использованный диапазон масштабов
   - Число смещений сетки
   - Критерии стабильности

### Для практического применения

1. **Используйте D как сравнительную метрику:**
   - Относительные изменения более значимы
   - Абсолютное значение требует интерпретации

2. **Комбинируйте с другими методами:**
   - Парадокс береговой линии
   - Фрактальная аппроксимация
   - Визуальная инспекция

3. **Учитывайте природу данных:**
   - Разрешение исходного контура
   - Географический масштаб
   - Природные процессы

## Заключение

Метод box-counting в реализации Litoro-cli представляет собой улучшенный алгоритм оценки фрактальной размерности с повышенной стабильностью и воспроизводимостью. Ключевые улучшения (усреднение по смещениям, плотный набор масштабов, адаптивный выбор окна) направлены на снижение методической вариативности и получение более надёжных оценок для реальных географических данных.
