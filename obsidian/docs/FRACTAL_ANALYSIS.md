# Фрактальный анализ

Фрактальный анализ предоставляет количественную оценку сложности береговой линии через эмпирическую фрактальную размерность, вычисленную методом box-counting с усреднением по нескольким смещениям сетки и адаптивной регрессией для обеспечения максимальной точности и устойчивости результатов.

## Обзор методологии

**Фрактальная размерность** (`D`) количественно характеризует насколько сильно кривая изгибается и заполняет плоскость. Для идеальных фракталов `D` постоянна, но для природных объектов она зависит от масштаба наблюдения.

**Диапазон значений:**
- `D = 1.0` — идеально гладкая линия (евклидова размерность)
- `D ≈ 1.05–1.15` — слабо изрезанная береговая линия
- `D ≈ 1.15–1.25` — умеренно изрезанная береговая линия (типичная)
- `D ≈ 1.25–1.35` — сильно изрезанная береговая линия (фиорды, шхеры)
- `D ≈ 1.2619` — кривая Коха (теоретическое значение)
- `D → 2.0` — кривая заполняет плоскость

**Архитектура модуля:**

```
internal/domain/fractal/
└── dimension.go          # Box-counting анализ
```

**Основные типы данных:**

```go
type BoxCountingSample struct {

    RelativeScale float64 // Относительный размер ячейки

    Valid              bool      // Валидность результата
}
```

## Box-counting размерность

Метод box-counting (подсчёт ячеек) является стандартным способом оценки фрактальной размерности кривых через анализ их покрытия квадратными ячейками различного размера.

### Математическая основа метода

**Степенной закон покрытия:**

Для фрактальной кривой число ячеек `N(ε)` размера `ε`, необходимых для покрытия кривой, подчиняется степенному закону:

```
N(ε) ∝ ε^(-D)
```

где:
- `N(ε)` — минимальное число ячеек размера `ε` для покрытия кривой
- `ε` — размер ячейки (scale)
- `D` — фрактальная размерность

**Логарифмическое преобразование:**

Логарифмируя выражение, получаем линейную зависимость:

```
ln(N(ε)) = D × ln(1/ε) + C

D = d[ln(N)] / d[ln(1/ε)]
```

Наклон линейной регрессии в координатах `ln(N)` от `ln(1/ε)` даёт оценку размерности `D`.

**Интерпретация для идеальных фракталов:**
- Для прямой линии: `N(ε) ∝ ε^(-1)`, следовательно `D = 1.0`
- Для кривой Коха: `D = ln(4)/ln(3) ≈ 1.26186`
- Для кривой, заполняющей плоскость: `D → 2.0`

### Проекция географических координат

Для вычисления расстояний в метрах географические координаты проецируются в локальную декартову систему:

```go
func latLonToMeters(p LatLon) Point2D {
    const (
        refLat          = 43.5         // Опорная широта (центр Чёрного моря)
        metersPerDegLat = 111194.9     // Метров в градусе широты
        metersPerDegLon = 87300.0      // Метров в градусе долготы на широте 43.5°
    )
    
    dLat := (p.Lat - refLat) * metersPerDegLat
    dLon := (p.Lon - 35.0) * metersPerDegLon
    
    return Point2D{X: dLon, Y: dLat}
}
```

**Bounding box:**

```
minX, maxX, minY, maxY = extrema(points)
width  = maxX - minX
height = maxY - minY
bboxSize = max(width, height)
```

`bboxSize` используется как базовый масштаб для формирования масштабного ряда.

### Масштабный ряд и дискретизация

**Плотный масштабный ряд:**

Для снижения чувствительности к произвольному выбору масштабов используется плотный набор из 13 масштабных факторов:

```go
var defaultScaleFactors = []float64{
    4, 6, 8, 12, 16, 24, 32, 48, 64, 96, 128, 192, 256,
}
```

**Вычисление размера ячейки:**

Для каждого фактора `s`:

```
boxSize = bboxSize / s
relativeScale = 1 / s = boxSize / bboxSize
```

**Пример для bboxSize = 1000 км:**

| Scale Factor | Box Size | Relative Scale |
|--------------|----------|----------------|
| 4 | 250 км | 0.25 |
| 8 | 125 км | 0.125 |
| 16 | 62.5 км | 0.0625 |
| 64 | 15.6 км | 0.0156 |
| 256 | 3.9 км | 0.0039 |

Диапазон от `bboxSize/256` (мелкие детали) до `bboxSize/4` (грубая структура).

### Усреднение по смещениям сетки

**Проблема:** Результат box-counting чувствителен к положению решётки относительно кривой. Неудачное положение может дать искажённые результаты.

**Решение:** Использование 4 смещений сетки с последующим усреднением:

```go
var gridOffsets = [][2]float64{
    {0, 0},     // Базовая сетка
    {0.5, 0},   // Сдвиг по X на пол-ячейки
    {0, 0.5},   // Сдвиг по Y на пол-ячейки
    {0.5, 0.5}, // Сдвиг по обоим осям
}
```

**Алгоритм усреднения:**

```go
func boxesCoveredMetersAverage(points, boxSize, minX, minY, offsets) float64 {
    sum := 0.0
    for _, offset := range offsets {
        covered := make(map[[2]int]struct{})
        for i := 1; i < len(points); i++ {
            markSegmentBoxesOffset(covered, points[i-1], points[i], 
                                   boxSize, minX, minY, offset[0], offset[1])
        }
        sum += float64(len(covered))
    }
    return sum / float64(len(offsets))
}
```

**Формула усреднения:**

```
N_avg(ε) = Σ N(ε, offsetᵢ) / 4,  i = 0..3
```

Это снижает артефакты, возникающие при неудачном положении сетки.

### Алгоритм покрытия сегментов

Для каждого отрезка кривой определяется множество ячеек, которые он пересекает:

```go
func markSegmentBoxesOffset(covered, a, b, boxSize, minX, minY, offsetX, offsetY) {
    dx := b.X - a.X
    dy := b.Y - a.Y
    distance := √(dx² + dy²)
    
    // Число шагов дискретизации (полу-ячейка для надёжности)
    steps = ceil(distance / (boxSize/2)) + 1
    steps = max(steps, 2)
    
    for i := 0..steps:
        t = i / steps
        x = a.X + dx × t
        y = a.Y + dy × t
        
        // Индексы ячеек с учётом смещения
        row = floor((y - minY + offsetX × boxSize) / boxSize)
        col = floor((x - minX + offsetY × boxSize) / boxSize)
        
        covered[(row, col)] = {}
}
```

**Ключевые моменты:**
- **Дискретизация с шагом `boxSize/2`** гарантирует, что ни одна ячейка не будет пропущена
- **Использование `map[(row, col)]struct{}`** автоматически устраняет дубликаты
- **Смещение `offsetX × boxSize, offsetY × boxSize`** реализует сдвиг сетки

**Сложность алгоритма:**
`O(n × m × k)`, где:
- `n` — число точек входной кривой
- `m` — число масштабных факторов (13)
- `k` — число смещений сетки (4)

## Статистическая устойчивость

Статистическая устойчивость результатов обеспечивается адаптивным выбором диапазона масштабов для регрессионного анализа и многокритериальной валидацией полученной размерности.

### Логарифмическое преобразование

Данные box-counting преобразуются для линейной регрессии:

```
X = ln(1 / relativeScale) = ln(scaleFactor)
Y = ln(N_avg(scaleFactor))
```

**Пример данных для регрессии:**

| Scale Factor | Box Size (km) | Boxes Covered | X = ln(s) | Y = ln(N) |
|--------------|---------------|---------------|-----------|-----------|
| 8 | 125 | 45 | 2.08 | 3.81 |
| 16 | 62.5 | 89 | 2.77 | 4.49 |
| 32 | 31.3 | 167 | 3.47 | 5.12 |
| 64 | 15.6 | 312 | 4.16 | 5.74 |

Наклон линейной регрессии `Y = a × X + b` даёт оценку размерности: `D = a`.

### Адаптивный выбор окна регрессии

**Проблема:** Использование всех масштабных точек может дать нестабильную оценку из-за отклонения от степенного закона на крайних масштабах.

**Решение:** Перебор всех возможных окон регрессии и выбор наилучшего по комплексному критерию:

```go
func bestRegressionWindow(x, y []float64) *regressionWindow {
    n := len(x)
    var best *regressionWindow
    
    for start := 0; start <= n - minScaleSamples; start++ {
        for end := start + minScaleSamples - 1; end < n; end++ {
            xs := x[start:end+1]
            ys := y[start:end+1]
            
            slope, intercept := linearRegression(xs, ys)
            r² := regressionRSquared(xs, ys, slope, intercept)
            localDims := localSlopeSeries(xs, ys)
            spread := valueSpread(localDims)
            
            stable := len(localDims) ≥ 3 ∧ r² ≥ 0.98 ∧ spread ≤ 0.18
            
            candidate := {start, end, length, slope, r², spread, stable}
            
            if betterWindow(best, candidate, stable):
                best = candidate
        }
    }
    
    return best
}
```

**Критерии выбора окна (приоритеты):**

```
1. Стабильность: стабильное окно > нестабильное
2. Длина: длинное окно > короткое (больше данных)
3. R²: более высокое R² > низкое
4. Наклон: больший наклон > меньший (при близких R²)
5. Разброс: меньший spread > больший
```

**Линейная регрессия (метод наименьших квадратов):**

```
         n × Σ(xy) - Σx × Σy
slope = ──────────────────────
           n × Σ(x²) - (Σx)²

           Σy - slope × Σx
intercept = ───────────────
                  n
```

**Коэффициент детерминации R²:**

```
         Σ(yᵢ - ŷᵢ)²
R² = 1 - ────────────
         Σ(yᵢ - ȳ)²

где ŷᵢ = slope × xᵢ + intercept
      ȳ = mean(y)
```

- `R² = 1.0` — идеальная линейная зависимость
- `R² < 0.98` — результат может быть ненадёжным

### Локальные размерности и разброс

**Локальные размерности** — наклоны между соседними точками в выбранном окне:

```go
func localSlopeSeries(x, y []float64) []float64 {
    slopes := []
    for i := 1; i < len(x); i++:
        if |x[i] - x[i-1]| < 1e-12:
            continue  // пропускаем вырожденные
        slopes.append((y[i] - y[i-1]) / (x[i] - x[i-1]))
    return slopes
}
```

**Разброс локальных размерностей:**

```
spread = max(localDimensions) - min(localDimensions)
```

**Интерпретация разброса:**

| Spread | Качество результата |
|--------|---------------------|
| `< 0.10` | Отличная стабильность — D почти не зависит от масштаба |
| `0.10–0.15` | Хорошая стабильность — D умеренно зависит от масштаба |
| `0.15–0.18` | Приемлемая стабильность — D заметно зависит от масштаба |
| `> 0.18` | Низкая стабильность — результат может быть артефактом |

### Критерии валидности результата

**Полный набор критериев валидности:**

```go
stable := len(localDimensions) ≥ 3          // Достаточно локальных оценок
        ∧ r² ≥ 0.98                         // Высокое качество регрессии
        ∧ spread ≤ 0.18                     // Приемлемый разброс

valid := stable ∧ slope ∈ [0.5, 3.0]        // Реалистичный диапазон размерности
```

**Реалистичный диапазон размерности:**
- `slope < 0.5` — кривая слишком гладкая (артефакты данных)
- `slope > 3.0` — кривая слишком сложная (возможно, ошибка в данных)

**Пример валидного результата:**

```go
BoxCountingAnalysis{
    Dimension:          1.2345,
    RegressionRSquared: 0.9876,
    StableAcrossScales: true,
    StabilitySpread:    0.12,
    Valid:              true,
}
```

**Пример невалидного результата:**

```go
BoxCountingAnalysis{
    Dimension:          0.0,        // Не вычислено
    RegressionRSquared: 0.9234,    // Недостаточно высокое
    StableAcrossScales: false,
    StabilitySpread:    0.25,      // Слишком большой разброс
    Valid:              false,
}
```

## Диагностика сходимости

Диагностика сходимости анализирует изменение фрактальной размерности по итерациям генерации фрактальных кривых (например, кривых Коха), позволяя оценить, насколько размерность приближается к теоретическому значению.

### Итерационный анализ размерности

**Принцип:** При генерации фрактальной кривой (например, итеративное построение кривой Коха) фрактальная размерность на каждой итерации должна стремиться к теоретическому значению.

**Алгоритм итерационного анализа:**

```go
for iteration := 0; iteration <= maxIterations; iteration++ {
    curve := generateFractal(base, iteration)
    
    analysis := AnalyzeBoxCounting(curve)
    
    dimensions[iteration] = analysis.Dimension
    valid[iteration] = analysis.Valid
    rSquared[iteration] = analysis.RegressionRSquared
}
```

**Структура данных для итерационного анализа:**

```go
type fractalIterationMetrics struct {
```

### Теоретические ожидания для кривых Коха

**Классическая кривая Коха:**

Теоретическая фрактальная размерность:

```
D_theoretical = ln(4) / ln(3) ≈ 1.261859507...
```

**Ожидаемая динамика по итерациям:**

| Итерация | Число точек | Ожидаемая D | Типичная ошибка |
|----------|-------------|-------------|-----------------|
| 0 | 2 | 1.0000 | 20.7% |
| 1 | 5 | 1.1200 | 11.2% |
| 2 | 17 | 1.2000 | 4.9% |
| 3 | 65 | 1.2400 | 1.7% |
| 4 | 257 | 1.2550 | 0.5% |
| 5 | 1025 | 1.2610 | 0.1% |

**Закон сходимости:** Ошибка уменьшается примерно экспоненциально с номером итерации.

### Критерии сходимости

**Сходимость к теоретическому значению:**

```go
func evaluateConvergence(dimensions []float64, theoretical float64) ConvergenceMetrics {
    errors := make([]float64, len(dimensions))
    
    for i, d := range dimensions {
        if d > 0 {
            errors[i] = math.Abs(d - theoretical) / theoretical * 100
        }
    }
    
    // Критерии сходимости
    finalError := errors[len(errors)-1]
    convergenceRate := errors[0] / finalError
    
    return ConvergenceMetrics{
        FinalError:       finalError,
        ConvergenceRate:  convergenceRate,
        Converged:        finalError < 1.0,  // < 1% ошибка
        Monotonic:        isMonotonicDecrease(errors),
    }
}
```

**Критерии успешной сходимости:**

1. **Монотонное приближение:** Ошибки должны убывать (или не возрастать)
2. **Финальная точность:** Ошибка < 1% для высоких итераций
3. **Стабильность:** Последние 3-4 итерации дают близкие значения

**Пример успешной сходимости:**

```
Итерация 0: D = 1.0000, ошибка = 20.7%
Итерация 1: D = 1.1200, ошибка = 11.2%
Итерация 2: D = 1.2000, ошибка = 4.9%
Итерация 3: D = 1.2400, ошибка = 1.7%
Итерация 4: D = 1.2550, ошибка = 0.5%
Итерация 5: D = 1.2610, ошибка = 0.1% ✓ Сошлось
```

## Практическое применение

### Анализ реальных береговых линий

```go
// Загрузка данных
points, _, err := coastline.LoadFromJSON("data/black-sea.json")

// Полный анализ
analysis := fractal.AnalyzeBoxCounting(points)

fmt.Printf("Фрактальная размерность: %.4f\n", analysis.Dimension)
fmt.Printf("Качество регрессии (R²): %.4f\n", analysis.RegressionRSquared)
fmt.Printf("Стабильность: %v\n", analysis.StableAcrossScales)
fmt.Printf("Разброс: %.4f\n", analysis.StabilitySpread)
```

**Типичные результаты для Чёрного моря:**
- `D ≈ 1.15–1.22` — умеренно изрезанная береговая линия
- `R² > 0.98` — высокое качество регрессии
- `StableAcrossScales = true` — результат надёжен
